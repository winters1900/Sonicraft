# 设计文档 · AI 语音绘图工具

> 七牛云暑期赛作品。本文记录：计划支持的指令能力、最终实现情况、未完成部分及原因，以及关键架构与设计决策。

## 零、本次迭代要点（更新记录）

本轮围绕「真正接通七牛语音」「纯语音交互」「绘图表现力」「容错与稳健性」四块推进：

1. **七牛语音三件套全部跑通**
   - **ASR**：七牛 ASR 为「服务端回拉音频」模式（`audio.url` 必填、访问不到 localhost），故后端先把录音上传**七牛对象存储 Kodo** 换公网 URL 再交识别；响应解析按实测取 `data.result.text`。webm/wav/mp3 均可。
   - **TTS**：修正请求契约为 `audio.voice_type` + `request.text`（七牛实测格式，非 OpenAI 的 input/voice）。
   - 关键陷阱：Windows 下 `curl` 会损坏中文 UTF-8，调试中文接口须用 Node/浏览器。
2. **纯语音交互**：移除文字输入模块（仅保留只读识别日志）；**持续聆听**——七牛引擎用 Web Audio 能量 VAD 自动断句、浏览器引擎连续识别，开口即画、无需逐句点击；后端配置七牛时默认即用七牛 ASR。
3. **绘图能力增强**
   - 新增 5 种基础图形：五角星 / 椭圆 / 多边形(N 边形) / 心形 / 弧形。
   - **组合/语义绘图**：`compose` 预设把「画一个笑脸/房子/太阳/树/花/雪人/小猫」一句话展开为多图元组合（单一撤销单元）。
   - **组合分组**：同组共享 `groupId`，可「选中房子 / 把笑脸放大」整体重选与变换，缩放绕组中心不散开。
4. **容错与稳健性**
   - **回声抑制**：TTS 播报期间静音麦克风，杜绝系统把自己播报当新指令的自激循环。
   - **「保存图片」真正触发 PNG 下载**。
   - 规则覆盖**口语线宽**（粗/细）与**填充切换**（实心/空心）；颜色表扩到 ~55 个并修正「具体色被短色词遮蔽」（深蓝/草绿不再退化成蓝/绿）。
5. **测试**：单测由 37 增至 **65** 项，全绿。

> 受限项：七牛**文生图(Kling)** 当前密钥返回 403（账号未开通图像生成权限），暂不可用；七牛**流式 ASR(WebSocket)** 为二进制/gzip 协议、需真机麦克风联调，留作后续专项。

## 一、设计目标

打造一款**纯语音控制**的绘图工具——用户不使用鼠标/键盘，仅凭语音完成绘图创作。围绕赛题三项核心诉求展开设计：

1. **指令理解的准确性与容错性**：口语化、模糊、带填充词/纠错的表达都能正确理解。
2. **语音到绘图的响应延迟**：常用指令需即时响应。
3. **复杂指令的拆解与执行**：一句话包含多个动作时能拆解为有序操作并依次执行。

## 二、整体架构

```
麦克风（持续聆听：七牛=能量VAD自动断句 / 浏览器=连续识别）
  │  音频
  ▼
语音识别(ASR)   七牛 ASR：录音→上传 Kodo 换公网 URL→/voice/asr(url) ┐
  │  文本                                                        ├─ 二选一，可切换
  ▼              浏览器 Web Speech（无密钥兜底，实时字幕）         ┘
指令解析(混合)
  ├─ 本地规则 RuleParser ── 命中 ─→ DrawCommand[] / ComposeCommand   （零网络延迟）
  └─ 未命中 ─→ 后端 /api/parse ─→ 七牛 LLM ─→ DrawCommand[]          （强容错）
  │
  ▼
命令执行 CommandExecutor ──→ CanvasEngine(状态/历史/渲染/分组/导出)
  │  反馈文案
  ▼
语音反馈(TTS)   七牛 TTS(/voice/tts) / 浏览器 SpeechSynthesis(兜底)
                └─ 播报期间经 ttsGate 静音麦克风（回声抑制）
```

**分层与目录**

| 层 | 位置 | 职责 |
| --- | --- | --- |
| 绘图引擎 | `frontend/src/engine` | 图形模型、渲染、选区、撤销/重做、导出 |
| 指令 DSL | `shared/commands.ts` | 前后端共享的结构化指令契约 + 校验 |
| 解析层 | `frontend/src/parser` | 规则快路径 + 混合编排(规则↔LLM) |
| 执行层 | `frontend/src/executor` | DSL → 引擎调用，指代解析、批量排布、反馈文案 |
| 语音层 | `frontend/src/voice` | ASR/TTS 引擎抽象与切换、麦克风控制、字幕 |
| 控制器 | `frontend/src/controller` | 文字/语音统一执行入口 + 日志 + 反馈 |
| 后端 | `backend/src` | 七牛 LLM/ASR/TTS 代理，密钥隔离 |

### 核心抽象：DrawCommand DSL

解析层与执行层之间以结构化指令数组解耦。**一句话可产出多条命令**，这是复杂指令拆解的载体。
规则解析与 LLM 解析产出同一套 DSL，执行层无需关心来源。详见 [`shared/commands.ts`](../shared/commands.ts)。

## 四、关键设计决策

1. **混合解析（延迟 vs 容错的折中）**——高频指令走本地正则，实测 <1ms 命中即时执行；
   仅当规则未命中或低置信度时才请求七牛 LLM（带超时降级）。兼顾“响应延迟”与“理解准确性/容错”。
2. **结构化 DSL 解耦**——规则与 LLM 输出同一套 `DrawCommand`，执行层单一实现；
   LLM 用 `response_format=json_object` + few-shot 强约束，并经 `isValidCommand` 校验过滤，防注入脏数据。
3. **复杂指令拆解**——规则按连接词（然后/接着/，/。）分段；LLM 由 prompt 要求按动作顺序拆成多条命令。
   `count`/`layout` 支持“三个圆排成一行”这类批量+排布语义。
4. **指代消解**——`SelectTarget` 支持 last/all/byType/byColor/byIndex；新建图形即选中，
   使“画个圆…再大一点…变成红色”这类连续指代自然衔接。
5. **健壮的引擎**——快照式撤销/重做，批量创建为单一撤销单元；命令式 API 便于单测（已 37 项单测覆盖）。
6. **降级与兜底**——ASR/TTS 均“七牛优先 + 浏览器原生兜底”；LLM 失败退回规则结果。
   保证**无七牛密钥也能完整演示**绘图链路，有密钥则启用七牛全套能力。

## 五、七牛云服务对接（实测契约）

| 能力 | 接口 / 请求体 | 说明 |
| --- | --- | --- |
| 大模型解析 | `POST {base}/chat/completions`（OpenAI 兼容） | 自然语言 → DrawCommand[]；`temperature:0` + `response_format:json_object` |
| 语音识别 | `POST {base}/voice/asr`，体 `{model:'asr',audio:{format,url}}` | **url 必填**（服务端回拉音频，不收内联 base64）；文本在 `data.result.text` |
| 语音合成 | `POST {base}/voice/tts`，体 `{audio:{voice_type,encoding:'mp3'},request:{text}}` | 返回 `{data:'<base64 mp3>'}`；音色 `GET {base}/voice/list` |
| 对象存储 | 七牛 **Kodo**（SDK `qiniu`） | ASR 前置：录音上传换公网 URL 供七牛回拉；识别后 best-effort 删除临时对象 |

`base = https://api.qnaigc.com/v1`，鉴权 `Authorization: Bearer <QINIU_API_KEY>`。
AI 密钥与 Kodo 的 AK/SK 是两套凭证，均只存后端 `.env`（`QINIU_API_KEY`、`QINIU_KODO_AK/SK/BUCKET/DOMAIN`），前端不接触。

> 联调陷阱：Windows 下 `curl -d` 会破坏中文 UTF-8 导致七牛 400/500（假象）；测中文接口用 Node/浏览器（`JSON.stringify` 正确编码）。七牛**文生图**当前密钥 403（账号未开通），暂不可接入。

## 六、响应延迟设计

- **本地规则快路径**：常用指令（画圆、变色、撤销、清空…）零网络往返，<1ms 解析后立即渲染。
- **LLM 仅用于复杂/模糊指令**，并设超时（默认 9s）降级为 unknown 触发追问，避免长时间无反馈。
- **ASR 字幕**：浏览器引擎实时显示中间结果（零延迟）；七牛路径含「上传 Kodo → 回拉识别」一跳，延迟略高于浏览器，适合短指令，演示低延迟可切浏览器引擎。
- 日志面板实时展示每条指令的「解析来源 + 耗时」，可直观验证延迟表现。

## 八、未完成部分及原因

| 项 | 原因 |
| --- | --- |
| 七牛文生图（Kling）入画 | 当前赛事密钥调 `/v1/images/generations` 返回 403（账号未开通图像生成权限）；待开通后再接入（后端加生成路由 + 引擎加 image 图元）。 |
| 七牛流式 ASR（WebSocket） | 流式接口为二进制/gzip 自定义分帧，需真机麦克风联调；REST + Kodo 已满足短指令，流式留作有真机环境的后续专项。 |
| 组合图形整组旋转 | 线段「锚点+终点+rotation」双表示使绕组中心旋转较复杂、风险高；移动/缩放/删除已整组生效，旋转留作 v2.1。 |
| 自由画笔/手绘曲线 | 语音难以精确描述连续笔迹轨迹，结构化图形性价比更高；列为非目标。 |
| 唤醒词激活 | 连续监听 + 一键开关已满足纯语音演示；唤醒词为锦上添花，优先级较低。 |
| 任意物体的自由组合（如“画一只小狗”） | 需 LLM 输出精确归一化坐标，温度 0 下方差大、风险高；确定性预设已覆盖常见物体，自由组合留作后续。 |

> 实现提示：七牛 ASR/TTS 的精确请求字段已按实测固化，集中在 `backend/src/routes/asr.ts`（含 Kodo 上传 `backend/src/kodo.ts`）与 `tts.ts`，便于按协议微调。
